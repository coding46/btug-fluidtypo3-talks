Automagische Controller + Actions

  • fluidpages ermöglicht PageController
  • fluidcontent ermöglicht ContentController
  • Beispiel fluidcontent_core

Regeln

  • fluidpages => PageController / ActionName = TemplateName
  • fluidcontent => ContentController / ActionName = TemplateName
  • PageController / ContentController / CoreContentController nicht anderweitig verwenden

Datenfluss

  • Automapping von Template-Variablen mit Prefix settings. auf $this->settings
  • Zusätzliche Variablen wie üblich per $this->view->assign( 'foo', $bar )
